DPB Overview

2019-09-07

DPB design aims to provide an extensible framework which abstracts the underlying infrastructure diversity. The architecture of a DPB orchestrator is presented in \fref{DPB-OverArch}. DPB follows a multi-layer design approach and consists of three key layers.

At the top of the system is the \defn{aggregator} layer, which is responsible for service persistence, allocation of trunk resources, and end-to-end path computation (\secref{aggregator}). The aggregator layer exposes a single northbound interface (NBI) to the service orchestrator for the management of logical networks across datacenters. In addition, the NBI of an aggregator can be used by a higher layer aggregator, enabling control delegation.

The \defn{logical-switch} layer (\secref{logical-switch}) constitutes the middle layer of the DPB architecture and is responsible for persisting configuration and forwarding state for network devices, presenting the same NBI as an aggregator. It interfaces with the \defn{fabric adaptation} layer below, which translates forwarding policy into technology-specific configuration. The communication between the different layers of DPB is facilitated by a hierarchical data model (\secref{model}).
